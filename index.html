WillJS

A great way to load assets on-demand, and a awesome way to develop components for webpages.

Download this project as a .zip file Download this project as a tar.gz file

Demo

Consider the following page:

<!DOCTYPE html>
<html>
<body>
<a href="javascript:will.call('popMessage')('Hello, this is a message on a dialog!');">
   click here!</a>
<script src="javascripts/will/will.min.js"></script>
</body>
</html>

Quite simple, right? There is nothing more than an external asset in this page (the will.min.js file). Note that the link element of this example has a weird calling, but still is possible to understand that a message will be displayed to the user by somewhere. Open the debugger of your browser to see what will happen. (after that, try it: click here)!

Rather Drive Stick?

If you prefer to not use the automatic generator, push a branch named gh-pages to your repository to create a page manually. In addition to supporting regular HTML content, GitHub Pages support Jekyll, a simple, blog aware static site generator written by our own Tom Preston-Werner. Jekyll makes it easy to create site-wide headers and footers without having to copy them across every page. It also offers intelligent blog support and other advanced templating features.
